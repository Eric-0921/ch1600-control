# CH-1600 数字高斯计 — 连接与控制指令参考

## 一、物理连接

### 串口规格

CH-1600 配备 RS-232C 串行接口（9 针 D 型母口，DCE 配线），同时具备 USB 接口。

| 项目 | 规格 |
|------|------|
| 连接器 | 9 针 D 型插座 (Female)，DCE 配线 |
| 传输模式 | 半双工 |
| 信号电平 | EIA RS-232C |
| 传输距离 | 最长 15 米 |
| 握手 | 无硬件握手 |
| 命令速率 | 最大每秒 10 命令 |

### 管脚映射

CH-1600 (DCE, DB-9F) 连接至 PC (DTE) 使用**直通电缆**：

| CH-1600 管脚 | 信号 | PC DB-9M 管脚 |
|:---:|------|:---:|
| 1 | 空 | - |
| 2 | Tx (发送) → | 2 (Rx) |
| 3 | Rx (接收) ← | 3 (Tx) |
| 5 | GND | 5 |
| 4,6,7,8,9 | 空 | - |

> CH-1600 为 DCE 设备，PC 为 DTE 设备。两者使用直通电缆即可连接。若两端同为 DCE 或同为 DTE，需使用空调制解调器交叉 Tx/Rx。

### 后面板接口

- **RS-232C**: 标准 9 针 D 型母口
- **USB**: USB 连接端口
- **BNC**: 监视模拟输出，±3.0V（对应 ±3.0T 或 ±30T）
- **探头**: 15 针 D 型连接器
- **工控端子**: 定制工控接口

## 二、字符格式

| 参数 | 值 |
|------|-----|
| 起始位 | 1 bit |
| 数据位 | 8 bits |
| 校验 | 无 |
| 停止位 | 1 bit |
| 命令结束符 | CR (0x0D) |
| 每字符位数 | 10 bits (1 + 8 + 1) |
| 波特率 | 19200 / 57600 / **115200** (默认) |

> 时序格式为异步 RS-232C，不支持硬件握手。数据帧以 `\n` (换行符) 为帧结束标志。

## 三、命令集

所有命令以 CR (0x0D, `\r`) 结尾，返回值以 `\n` 结尾。命令字符串使用 ASCII 编码。

### 3.1 数据采集命令

| 命令 | 功能 | 返回值 |
|------|------|--------|
| `DATA?>` | 开始实时数据流 | 连续发送数据帧 |
| `DATAC>` | 停止实时数据流 | 无 |
| `DATAS>` | 单次查询数据 | `ACK` + 实时数据 |

### 3.2 状态查询命令

| 命令 | 功能 | 返回值 |
|------|------|--------|
| `UNIT?>` | 查询当前单位 | `mT` / `Gauss` / `A/m` / `oe` |
| `RANGE?>` | 查询当前量程 | `30—` / `300—` / `3k—` / `Auto` |
| `UPHRES?>` | 查询上限阈值 | `±/-xxxx.xx` (mT) |
| `LOWTHRES?>` | 查询下限阈值 | `±/-xxxx.xx` (mT) |

### 3.3 控制命令

| 命令 | 功能 | 返回值 |
|------|------|--------|
| `UNITSET>` | 切换单位（循环: mT→Gauss→A/m→Oe） | 无 |
| `RANGESET>` | 切换量程（循环: 30mT→300mT→3T→Auto） | 无 |
| `ZERO>` | 归零（将当前读数设为零点） | 无 |
| `MAX_MIN>` | 显示最大/最小值 | 无 |
| `LOCK>` | 界面锁定 | 无 |
| `UNLOCK>` | 解除锁定 | 无 |
| `RELA>` | 清零后恢复原显示值 | `ACK` |

### 3.4 阈值设置命令

| 命令 | 功能 | 返回值 |
|------|------|--------|
| `UPTHRES+xxxx.xx>` | 设置上限阈值 | 无 |
| `UPTHRES-xxxx.xx>` | 设置上限阈值 (负值) | 无 |
| `LOWTHRES+xxxx.xx>` | 设置下限阈值 | 无 |
| `LOWTHRES-xxxx.xx>` | 设置下限阈值 (负值) | 无 |

> 阈值可精确到 0.01 mT。超出阈值范围时设备发出声音报警并在屏幕显示 `(!)` 提示。

## 四、实时数据格式

### DATA?> 实时流数据帧

**格式**：
```
#±xxxxx.xxxx/xxx/±xxxx>\n
```

**字节长度**: 23~24 字节/帧

**字段**：
| 位置 | 字段 | 说明 | 示例 |
|------|------|------|------|
| 1 | `#` | 帧起始标志 | `#` |
| 2-11 | 磁场值 | ±xxxxx.xxxx mT | `-12345.6789` |
| 12 | `/` | 分隔符 | `/` |
| 13-15 | 频率 | xxx Hz (直流为 `000`) | `050` |
| 16 | `/` | 分隔符 | `/` |
| 17-20 | 温度 | ±xxx (实际值×10) | `+0234` |
| 21 | `>` | 帧结束标志 | `>` |
| 22 | `\n` | 换行符 | LF |

**示例**：
```
#-12345.6789/050/+0234>\n
```
表示：磁场值 = -12345.6789 mT，频率 = 50 Hz，探头温度 = +23.4°C

### DATAS> 单次查询数据

**格式**：
```
ACK#X轴数值/X轴频率/X轴温度;Y轴数值/Y轴频率/Y轴温度;Z轴数值/Z轴频率/Z轴温度>\n
```

> 温度值为实际值乘以10（例如 `+0256` = +25.6°C）。一维模式仅返回 X 轴数据。

## 七、探头信息

- 标配：Model-HCHD801F 超高精度数字化超薄横向霍尔探头
- 量程：10T (100 KGs)
- 探头内含非易失存储器，存储校准数据
- 同型号探头可直接替换，无需重新校准
- **连接前必须关闭设备电源**（上电后连接可能导致存储器失效）
- 预热：至少 5 分钟，10 分钟后达到额定精度
## 2026-05-21 Field Note: Tested CH-1600 Command Framing

The real CH-1600 connected on `COM13` and the decompiled vendor/debug program
show that this hardware expects raw ASCII command frames. Do not add CR/LF for
this tested path.

Validated examples:

- Start normal stream: `DATA?>`
- Stop stream: `DATAC>`
- Zero: `ZERO>`
- High speed: `FAST2>`, `FAST050>`, `FAST100>`, `FAST150>`, `FAST200>`,
  `FAST250>`, `FAST300>`

Important anti-regression notes for agents:

- `DATA?>` responded in live validation; `DATA?>\r` did not.
- 20 Hz is `FAST2>` for the tested CH-1600, not `FAST020>`.
- High-speed frames can be single-value frames like `#+0000.1433>`.
- Do not use this reference to reintroduce CR terminators or `FAST020>` unless
  a new live-device trace proves that a different hardware/firmware revision
  needs it.
