# 协议与采集逻辑发现

以下结论来自 `decompiled/数据读取软件/datapick/Form1.cs` 的静态反编译结果。

## 串口命令

| 场景 | 命令 |
| --- | --- |
| 普通数据启动 | `DATA?>` |
| 停止采集/关闭前停止 | `DATAC>` |
| 归零 | `ZERO>` |
| 查询类型 | 先 `DATAC>`，再 `TYPE>` |
| 高速 20 组/s | `FAST2>` |
| 高速 50 组/s | `FAST050>` |
| 高速 100 组/s | `FAST100>` |
| 高速 150 组/s | `FAST150>` |
| 高速 200 组/s | `FAST200>` |
| 高速 250 组/s | `FAST250>` |
| 高速 300 组/s | `FAST300>` |

注意：原程序里 20 组/s 使用的是 `FAST2>`，不是 `FAST020>`。这不是只在某个 1D 型号分支里出现，而是高速下拉框的通用映射。

## 高速启动差异

- 对 `CH-1500` / `CH-1500B`：发送 `DATA?>` + `Speed` + `NORM>`，例如 `DATA?>FAST050>NORM>`。
- 对 `CH-1600` 等其他高速型号：只发送 `Speed`，例如 `FAST2>`、`FAST100>`。
- 普通模式下发送 `DATA?>`。
- 停止统一发送 `DATAC>`。

## 型号与帧长度

| 型号 | 普通帧读取长度 `ReadLength` | 高速帧基长 `HighSpeedBase` | 高速数据 |
| --- | ---: | ---: | --- |
| `CH-1300` | 20 | 默认 13 | UI 禁用 |
| `CH-1500` | 21 | 11 | 可用 |
| `CH-1500B` | 默认 23 | 默认 13 | 可用 |
| `CH-1600` | 默认 23 | 默认 13 | 可用 |
| `CH-1800` | 24 | 14 | 可用 |
| `CH-260` / `CH-290` | 24 | 14 | UI 禁用 |
| 手动磁通/磁强型号 | 24 或默认 | 14 或默认 | UI 禁用 |

## 普通数据帧

普通模式 `DataType_flag == 1`：

- 串口一次读取 `ReadLength` 个字符。
- 帧必须 `StartsWith("#")` 且 `EndsWith(">\n")`。
- 去掉起始 `#`、末尾 `\n`、末尾 `>`。
- 按 `/` 切分为 3 段：
  - 第 0 段：磁场/磁通数值
  - 第 1 段：频率 Hz
  - 第 2 段：温度，原程序除以 10 得到摄氏度

普通高斯计帧逻辑形态：

```text
#{field}/{freq}/{temp_x10}>\n
```

根据 `CH-1600` 的 `ReadLength = 23` 推断，典型固定宽度可能类似：

```text
#+000.0000/00000/0250>\n
```

这是根据长度和解析逻辑推断，后续可用串口监听验证。

## 高速数据帧

高速模式 `DataType_flag == 2`：

- 每批读取 `HighSpeedNO * HighSpeedBase` 个字符。
- 按 `\n` 分割。
- 每行长度必须等于 `HighSpeedBase - 1`。
- 每行去掉 `#` 和 `>` 后只解析磁场/磁通值。
- 高速模式不解析频率和温度，表格中写入 `0, 0`。

因此高速数据更像是纯数值流：

```text
#{field}>\n
#{field}>\n
#{field}>\n
```

## TYPE 查询

当 `FlagEnquire == 1` 时，程序读取 5 字符：

- `WEAK\n`：识别为弱磁高斯计，切到 `comboBoxSource.SelectedIndex = 2`
- `NORM\n`：识别为普通高斯计，切到 `comboBoxSource.SelectedIndex = 1`

## 单位换算

程序内部基础值按 mT 处理，显示层做换算：

- `mT`: 原值
- `G` / `Oe`: `mT * 10`
- `A/m`: `mT * 795.77`
- `T`: `mT / 1000`
- 弱磁：
  - `μT`: 原值显示
  - `mG`: `μT * 10`
- 磁通计：
  - `mWb` 原值
  - `kMx`: `mWb * 100`

## USB HID 线索

`CH-1600` / `CH-1800` 支持 USB 选项。HID 查找条件：

- VendorID: `1155`
- ProductID: `22352`
- 读取报告长度：65 字节
- 启动：`senddata[1] = 17`
- 停止：`senddata[1] = 18`
- 归零：`senddata[1] = 32`

USB 数据解析：

- 磁场 float：`m_rd_data[4..7]`，小端 float
- 频率：`m_rd_data[9] + m_rd_data[10] * 256`
- 温度：`(m_rd_data[12] + m_rd_data[13] * 256) / 10`
- 温度大于 900 时按 `-(temp - 1000)` 转负温度

## 和现有 Python 项目需要重点核对的差异

- 原程序高速 20 Hz 命令是 `FAST2>`，不是 `FAST020>`。
- 原程序还支持 `FAST150>` 和 `FAST250>`。
- 原程序对 `CH-1600` 高速启动只发 `FASTxxx>`，不是先发 `DATA?>`。
- 原程序停止命令是 `DATAC>`。
- 普通数据帧包含磁场/频率/温度三段，高速帧只有磁场值。
